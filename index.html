What?

Where?

The plan

I'm most familiar with Perl, having used it for the majority of my sysadmin life, so I'm starting there, brushing up on it. I have a couple of projects to try with it, so that's a good start.

From there, I'd like to learn Go, and Swift. I was introduced to Go by a co-worker, and it really looks great, and I think that it's got a good chance of gaining traction as a server-side language. With any luck, it'll supplant javascript and node in the near future. As for Swift, we use it at my place of work, and I've got ideas for my own apps that I want to pursue.

Once I've got my Perl skills back to something resembling expertise, I'll start building actual projects. I'll mix in Swift next, probably, and start in on my iPad app ideas. Finally I'll add Go to the mix, so that I can build the back ends that I need.
